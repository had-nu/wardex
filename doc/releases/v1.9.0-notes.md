## Wardex v1.9.0 — The Trust Release

A major milestone for Wardex, introducing the **Trust Store** and **Sealed Configuration** (wexstate), transforming Wardex from a reporting tool into a cryptographically verifiable security governance platform.

### New Features

#### 🛡️ Trust Store & Sealed Config
Wardex now ensures that the configuration governing your release gates (like `risk_appetite`) is signed and tamper-proof.
- **`wardex keygen`**: Generate ed25519 keypairs for operators.
- **`wardex trust`**: Manage a centralized `wardex-trust.yaml` (init, add, revoke) to define authorized actors and their roles (`admin`, `ciso`, `analyst`).
- **`wardex config seal`**: Authorized CISOs or Admins can seal a draft `wardex-config.yaml` into a `.wexstate` file.
- **Integrity Enforcement**: `wardex evaluate` now verifies the signature of the `.wexstate` and checks the signer's status in the Trust Store before making any decisions.

#### 🔑 RBAC for Profiles
Risk profiles can now be restricted to specific actors. The `allowed_actors` field in the configuration ensures that sensitive risk overrides (e.g., for emergency hotfixes) can only be invoked by authorized individuals, with identity verified via `WARDEX_ACTOR` in trusted environments.

#### 📈 Signed EPSS Enrichment
Support for cryptographically signed EPSS (Exploit Prediction Scoring System) enrichment files. This allows organizations to ingest and verify exploit probability data from trusted sources, preventing tampering with risk scores.

### Infrastructure & CLI
- **New commands**: `trust`, `config seal`, `keygen`.
- **Exit Codes**: Updated to distinguish between Gate Blocks (`10`), Compliance Failures (`11`), and Integrity/Seal failures (`3`).
- **Remote Trust**: Support for `WARDEX_TRUST_STORE` environment variable to reference a central governance repository (e.g., via Raw GitHub URLs).

### Fixed & Improved
- Redesigned CLI banner.
- Improved validation for keyring file permissions (enforcing `0400` or `0600`).
- Enhanced logging for RBAC violations and signature verification.

